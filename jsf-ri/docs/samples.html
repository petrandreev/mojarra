This release of JavaServer Faces technology includes these four examples:
cardemo
An online car store. This application is a proof-of-concept that demonstrates JavaServer Faces can be used to build big, robust Web applications.
 
components
A set of custom components. This application shows how to build various custom components with JavaServer Faces technology.
 
guessNumber
Duke is thinking of a number. Can you guess it? The simplest application, which shows a small set of standard components, page navigation, and validation.
 
non-jsp
A simple application without JSP pages. This application demonstrates that you can build JavaServer Faces applications that don't use JSP pages.

Running the Sample Applications

The JWSDP 1.2 is set up so that all of the sample applications are automatically deployed once you start your server. To run one of the applications:
  1. Start your server.
  2. Point your browser to http://localhost:8080.
  3. Locate the heading, JSF Samples on the page that opens.
  4. Click the link, located under the heading JSF Samples, corresponding to the example you want to run.

Building and Running the Sample Apps With Ant

In order to run one of the sample applications using the supplied Ant build.xml files, simply follow these steps after you've installed the Java Web Services Developer Pack, v. 1.2:

  1. On a system running the Solaris or Linux operating system, go to the <JWSDP_HOME>/bin directory and execute the catalina.sh script to bring up the Java WSDP. On a system running Microsoft Windows, from the Start menu, select Programs, Java(tm) Web Services Developer Pack 1.2, and Start Tomcat.

  2. Once the server is up and running, point your browser to http://localhost:8080, the default port at which the process is running. The page that is displayed contains links to several sample programs and administration tools.
  3. Click on one of the links to run the corresponding example.

Building and Running the Sample Apps Manually

It is also possible to build each of the sample apps manually rather than use the supplied Ant build.xml files. For example, to build and run example/cardemo, follow these steps:
For Solaris/Linux:
Note: the specific details for each sample app may differ
  1. % export JAVA_HOME=<your J2SE installation directory>
  2. % export JWSDP_HOME=<your JWSDP1.1 installation directory>
  3. % export JAXB_HOME=$JWSDP_HOME/jaxb-1.0
  4. % export JAXB_LIBS=$JAXB_HOME/lib
  5. % export JAXP_LIBS=$JWSDP_HOME/jaxp-1.2.2/lib
  6. % export JWSDP_LIBS=$JWSDP_HOME/jwsdp-shared/lib
  7. % export PATH=$JAXB_HOME/bin:$JWSDP_HOME/jwsdp-shared/bin:$PATH
  8. % cd $JAXB_HOME/examples/users-guide/SampleApp1
  9. % $JAXB_HOME/bin/xjc.sh po.xsd -p primer.po
  10. % export CLASSPATH=$JAXB_LIBS/jaxb-api.jar:$JAXB_LIBS/jaxb-ri.jar:$JAXB_LIBS/jaxb-xjc.jar: $JAXB_LIBS/jaxb-libs.jar:$JAXP_LIBS/jaxp-api.jar:$JAXP_LIBS/endorsed/xercesImpl.jar: $JAXP_LIBS/endorsed/xalan.jar:$JAXP_LIBS/endorsed/sax.jar:$JAXP_LIBS/endorsed/dom.jar: $JWSDP_LIBS/jax-qname.jar:$JWSDP_LIBS/namespace.jar:.
    Be sure to enter the above command on a single line.
  11. % $JAVA_HOME/bin/javac Main.java primer/po/*.java primer/po/impl/*.java
  12. % $JAVA_HOME/bin/java Main
