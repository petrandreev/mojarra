Installing JavaServer Faces Technology in JWSDP

  1. Download the JavaTM Web Services Developer Pack 1.3 (Java WSDP).
  2. Follow the Installation Instructions for the Java WSDP.
  3. Update your path so that JWSDP_HOME points to the location of your Java WSDP installation.
  4. Download the JavaServer Faces Beta ZIP file.
  5. Archive or remove the the old JavaServer Faces EA3 release located in JWSDP_HOME/jsf
  6. Unzip the JavaServer Faces Beta ZIP file into JWSDP_HOME and rename the base directory to "jsf"
  7. You should now have the contents of the JavaServer Faces Beta release in your JWSDP_HOME/jsf directory
  8. If you have previously run JavaServer Faces applications, you should remove your web server's cached files before running the new release. The cached files will be located in JWSDP_HOME/work/Catalina/localhost/jsf-*. You should remove all the old jsf-* files and directories before restarting your server.
You are now ready to start developing and running JavaServer Faces applications.
 

Running the Examples in JWSDP

To run one of the examples,
  1. Copy the desired WAR file from the example directory of the JavaServer Faces installation to JWSDP_HOME/webapps.

  2. Go to the  JWSDP_HOME/bin directory.

  3. In a terminal window start the web server.

    If you are on a Unix platform, invoke the following command to start the web server:

    catalina.sh run 

    If you are running Windows, invoke the following command to start the web server:

    catalina run


  4. Wait for the copyright license to display in your terminal window.

  5. Launch your browser.

  6. You can find a page containing links to the JavaServer Faces example programs on your default server port:
http://localhost:8080

You can also run each example individually by specifing the name of the war file in your web browser:

http://localhost:8080/<EXAMPLE NAME>/ 
For example, this is the address you enter if you are running the cardemo:

http://localhost:8080/jsf-cardemo/

Please note that the examples are provided as a proof-of-concept rather than a best-practices guide for writing JavaServer Faces applications.

Building and Running the Examples

To build one of the examples, you need to run its build.xml file using Ant , which is a portable make tool.

When Ant builds the application, it copies files to a temporary build directory. This directory contains an exact image of the binary distribution for your JavaServer Faces application. When Ant builds the WAR file, it packages all the files from the build directory into the WAR file while preserving the directory structure contained in the build directory

To build the examples you first need to enter some paths in the build.properties file:

  1. Make sure you have the necessary preconditions as listed in the ReleaseNotes.
  2. Make sure you have Ant version 1.5.1 installed and working on your machine.
  3. Copy build.properties.sample to build.properties and edit accordingly for your system.
  4. Go to the directory of the example you want to build.

To build an example WAR file and deploy it:

  1. Go to the directory of the example you want to build.
  2. At the command line, run the default target: 
    	ant
  3. The war file of the example you just built will be placed in the JWSDP_HOME/jsf/samples directory.
  4. In JWSDP 1.3 your application will be automatically deployed.
    • JWSDP 1.3 includes an XML file for each example application in the JWSDP_HOME/conf/Catalina/localhost directory. This file causes that application to be automatically deployed when you start Tomcat.
    • You can manually deploy an application by copying the war file you just generated to the JWSDP_HOME/webapps directory.
      • When manually deploying under JWSDP 1.3 make sure to remove the corresponding xml file located in the JWSDP_HOME/conf/Catalina/localhost directory.
  5. Start the server by either running catalina.sh run if you are using UNIX or catalina run if your are using Windows.
  6. When the server has started, open your browser and enter this URL: 
    	http://localhost:8080/<EXAMPLE NAME>
In JWSDP, you can also find a page containing links to the JavaServer Faces examples on your default server port:
	http://localhost:8080


To learn how to develop, build, and run your own JavaServer Faces application, please refer to the
JavaServer Faces Tutorial.


Installing, Building and Running the Examples in a Non-JWSDP container

  1. Download the JavaServer Faces Beta ZIP file
  2. Unzip the Zip file
  3. Copy build.properties.sample file to build.properties and edit accordingly for your system configuration.
    • Be sure to uncomment the build.standalone property in the build.properties file
  4. Make sure the following properties in your customized build.properties file point to the correct jar file location according to your system configuration:
    • tomcat-home
      • path to your web container
    • jsf-api.jar
      • available in the lib directory of your JavaServer Faces unzipped bundle
    • jsf-impl.jar
      • available in the lib directory of your JavaServer Faces unzipped bundle
    • commons-beanutils.jar
      • BeanUtils 1.6+ jar file available from Jakarta Commons
    • commons-digester.jar
      • Digester 1.5+ jar file available from Jakarta Commons
    • commons-logging.jar
      • Logging 1.0.3+ jar file available from Jakarta Commons
    • commons-collections.jar
      • Collections 2.1+ jar file available from Jakarta Commons
    • jsp.jar
      • JSP 1.2+ API jar available from Jakarta Tomcat
    • jstl.jar
      • JSTL 1.0+ API jar file available from Jakarta Taglibs
      • You should use JSTL 1.1 in a container that supports JSP 2.0. If your container does not support JSP 2.0 then use JSTL 1.0.x.
      • The JavaServer Faces standalone examples contain JSTL 1.1 implementation jars.
    • standard.jar
      • JSTL 1.0+ Implementation jar file available from Jakarta Taglibs
      • You should use JSTL 1.1 in a container that supports JSP 2.0. If your container does not support JSP 2.0 then use JSTL 1.0.x.
      • The JavaServer Faces standalone examples contain JSTL 1.1 implementation jars.
    • servlet.jar
      • Servlet 2.3+ API jar available from Jakarta Tomcat
  5. If you have previously run JavaServer Faces applications, you should remove your web server's cached files before running the new release. The cached files will be located in JWSDP_HOME/work/Catalina/localhost/jsf-*. It is safe to remove all these files before restarting your server.
  6. Follow the Building and Running the Examples section
You will need to rebuild each of the JavaServer Faces example applications with the build.standalone property set to "true" so that each example's web archive includes all of the jar file dependencies. This step is not necessary in a JWSDP container since all the jar dependencies that JavaServer Faces requires are already part of the container.

Note: the Building and Running the Examples section makes reference to JWSDP_HOME, substitute your container's home directory for JWSDP_HOME. You will also need to manually deploy the example war files into your container's webapps directory
