JavaServerTM Faces 1.0 EA1
Release Notes


Revised:01/03/2002

The purpose of this release is to introduce you to the JavaServer Faces framework and to provide you with an opportunity to give us feedback on the work we've done so far.

This Early Access release of JavaServer Faces provides an early alpha version of the JSF technology which is currently being developed under the JCP as JSR127. The expert group is actively working on the specification and has not yet released the Community or Public drafts. This alpha release reflects a very early snapshot of a draft specification which has evolved significantly since the code freeze for this EA release.  This means that the APIs and designs contained in this release do not necessarily match what will be released in the upcoming Community and Public drafts of the specification.  This should be kept in mind when experimenting with this JSF technology.

What This Release Includes

     
  • html_basic.tld: A JSP custom tag library for building JSF applications that render to an HTML client.
  • An API and reference implementation that provide tag handling, object management, rendering, and event processing.
  • JavaServer Faces, a tutorial to help you get started.

Current Limitations

This release is limited to the following pieces of JSF functionality:

  • UI Components (Form, Command, SelectBoolean, SelectOne, TextEntry, Output)
  • JSP Tag Library for building JSF applications
  • ObjectTable for managing objects in scoped namespace
  • Rendering Model (and concrete RenderKit for standard HTML rendering)
  • Event Model


The following pieces of functionality will be part of the JSF specification but are *not* included in this release:

  • Some UI Components (Graphic, Panel, DataGrid, DataHierarchy)
  • Validation
  • Page Flow & Error Handling
  • I18N
  • Application lifecycle events

Additionally, in the areas of functionality released in the EA, there are already significant changes in the specification with respect to the component classes, model object management, and the event model. Therefore, only experimental code should be developed against this release since it will not be compatibile with subsequent releases.
